# /shit — 知识代谢技能

eat 负责吸收，shit 负责排泄。定期清理才能保持系统轻量、无冲突。

## 工作流（四步，顺序执行）

### Step 1：全量扫描

读取以下位置，建立完整清单：

```
~/.claude/skills/*/SKILL.md          # 所有 Skill（读 name + description）
~/.claude/CLAUDE.md                  # 全局指令
~/.claude/rules/*.md                 # 全局规则
~/.claude/projects/*/memory/*.md     # 项目记忆
```

对每个条目记录：名称、用途、最后修改时间。

### Step 2：诊断分析

按以下维度检查每个条目：

| 症状 | 判断标准 |
|-----|---------|
| **过时** | 引用的工具/API/路径已不存在；描述的流程已被新方式取代 |
| **冗余** | 两个 Skill 功能重叠度 > 80%；规则在多处重复定义 |
| **冲突** | 两条规则互相矛盾（如一个说"用 A"，另一个说"不用 A"） |
| **膨胀** | description 模糊导致误触发；内容过长但核心逻辑只有几行 |
| **孤儿** | Memory 引用的项目/文件已不存在 |

### Step 3：清理建议

给出具体方案，按优先级排列：

| 操作 | 适用情况 |
|-----|---------|
| **删除** | 完全过时或被取代，无保留价值 |
| **合并** | 两个 Skill/Rule 合为一个，消除冗余 |
| **精简** | 内容臃肿，瘦身到核心逻辑 |
| **修正** | 存在冲突，保留正确的那条，修改或删除错误的 |
| **保留** | 检查通过，无需处理 |

输出格式：
```
💩 代谢报告

扫描范围：X 个 Skill / Y 条 Rule / Z 条 Memory

建议清理：
1. [删除] skills/xxx — 原因：已被 yyy 完全取代
2. [合并] skills/aaa + skills/bbb → skills/ccc — 原因：功能重叠 90%
3. [精简] rules/ddd.md — 原因：200 行可压缩到 30 行
4. [修正] CLAUDE.md 第 N 行与 rules/eee.md 冲突

保留（健康）：
- skills/fff ✓
- skills/ggg ✓

是否执行？可逐条确认或全部执行。
```

### Step 4：执行清理

用户确认后执行。支持：
- 全部执行
- 逐条确认（y/n/skip）
- 只执行某几条（如 "执行 1 和 3"）

执行时：
- 删除前备份到 `/tmp/shit-backup-<date>/`，防止误删
- 合并时保留内容更完整的那份为基础
- 修改完成后汇报变更摘要

---

## 何时主动提醒

以下情况下，即使用户没有调用 /shit，也可以建议跑一次：

1. Skill 总数超过 50 个
2. /eat 执行时发现大量冲突或重叠
3. 用户反馈"某个 Skill 不该触发"或"规则好像矛盾了"

## 输出规范

- 用中文输出（除代码和文件路径）
- 建议阶段不动文件，等用户确认
- 删除操作必须先备份
- 执行后汇报变更摘要
